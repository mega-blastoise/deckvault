# Deploying

Deployments are manual. Images are built from the monorepo root on the Spark and loaded directly into Docker. No registry push is required for local deployments.

For per-service rollouts, see [Rolling Out a Service](/rollout-services).

## Full Deployment (all services)

### 1. Build images

Run from the monorepo root. The build context is the repo root so each Dockerfile can `COPY` from the workspace it needs.

```bash
docker build -t ghcr.io/mega-blastoise/deckvault-web:latest \
  -f apps/web/Dockerfile .

docker build -t ghcr.io/mega-blastoise/deckvault-rest-api:latest \
  -f apps/rest-api/Dockerfile .

docker build -t ghcr.io/mega-blastoise/deckvault-graphql-api:latest \
  -f apps/graphql-api/Dockerfile .
```

### 2. Copy updated compose file (if changed)

```bash
sudo cp docker-compose.prod.yml /opt/deckvault/docker-compose.prod.yml
sudo chown deckvault:deckvault /opt/deckvault/docker-compose.prod.yml
```

### 3. Restart via systemd

```bash
sudo systemctl restart deckvault
sudo systemctl status deckvault
```

Expected status: `active (exited)` with `Result: success`. Then verify all containers are up:

```bash
docker ps --format "table {{.Names}}\t{{.Status}}" | grep pokemon
```

Expect all six containers `Up` with `rest-api`, `graphql-api`, and `postgres` marked `(healthy)`.

### 4. Smoke test

```bash
curl -sI https://deckvault.gg/nginx-health | head -3
# Expect: HTTP/2 200, server: cloudflare

curl -s https://deckvault.gg/api/v1/health
# Expect: {"status":"ok"} or similar
```

## Rollback

Docker does not retain previous `:latest` images once overwritten. For rollback:

1. Check out the previous commit in git
2. Rebuild the affected image(s)
3. Restart via systemd

If you need a reliable rollback target, tag images explicitly before overwriting `:latest`:

```bash
docker tag ghcr.io/mega-blastoise/deckvault-web:latest \
  ghcr.io/mega-blastoise/deckvault-web:pre-deploy-$(date +%Y%m%d)
```

## Environment File

Secrets are stored in `/opt/deckvault/.env.prod` (chmod 600, owned by `deckvault`). This file is never committed to the repository. Update it directly on the Spark if environment variables need to change, then restart the affected service.

```bash
sudo nano /opt/deckvault/.env.prod
sudo systemctl restart deckvault
```

## nginx Config Changes

nginx config is mounted from the repo directory at `/home/nicks-dgx/dev/.Project-Johto/Pokemon/docker/nginx/`. Editing the file on disk takes effect on the next container restart — no copy to `/opt/deckvault/` is needed since nginx mounts it directly from the repo.

```bash
# After editing docker/nginx/conf.d/default.conf:
docker restart pokemon-nginx
```
