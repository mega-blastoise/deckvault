# Deployment Overview

DeckVault runs self-hosted on an NVIDIA DGX Spark, routed through a Cloudflare Tunnel. Six Docker containers are orchestrated by `docker-compose.prod.yml`, managed by a systemd unit. Images are built locally and loaded directly — no registry push/pull required for local deployments.

## Stack at a Glance

| Component | Technology |
|-----------|-----------|
| Host | NVIDIA DGX Spark (self-hosted) |
| Inbound traffic | Cloudflare Tunnel (`cloudflared`) |
| TLS | Cloudflare edge (terminates before nginx) |
| DNS | Cloudflare (`ben.ns.cloudflare.com`, `elisabeth.ns.cloudflare.com`) |
| Reverse proxy | nginx 1.27-alpine (HTTP-only, internal) |
| Orchestration | docker compose, managed by systemd |
| User data | PostgreSQL 17 (Docker named volume `pokemon_postgres_data`) |
| Card data | SQLite read-only volume (`/opt/deckvault/database/`) |
| Container registry | GHCR (`ghcr.io/mega-blastoise/`) |

## No Git on Prod

The `/opt/deckvault/` directory has no git checkout. It contains only five files:

```
/opt/deckvault/
├── docker-compose.prod.yml   # copied from repo when changed
├── .env.prod                 # secrets (chmod 600, never committed)
└── database/
    └── pokemon-data.sqlite3.db
```

Config changes are deployed by copying the updated file to `/opt/deckvault/` and restarting the systemd unit. The image is the single source of truth for application code.
