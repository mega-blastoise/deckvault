# Deployment Overview

DeckVault runs on a single EC2 instance behind CloudFront and Route53. Four Docker containers are orchestrated with `docker compose` and images are distributed via GHCR.

## Stack at a Glance

| Component | Technology |
|-----------|-----------|
| Compute | EC2 `t4g.small` (arm64, Amazon Linux 2023) |
| CDN | CloudFront |
| DNS | Route53 |
| Database | RDS PostgreSQL |
| Container registry | GHCR (`ghcr.io/mega-blastoise/`) |
| Reverse proxy | nginx:1.27-alpine |
| Orchestration | docker compose (prod profile) |

## No Git on EC2

The EC2 instance has no git checkout. Deployment works by:

1. Building images locally
2. Pushing to GHCR
3. SCPing any changed compose/config files
4. SSHing in and running `docker compose pull && up -d`

This keeps the instance lean and ensures the image is the single source of truth.
