# Security

## Cloudflare

TLS terminates at Cloudflare's edge. All traffic to `deckvault.gg` passes through Cloudflare before reaching the Spark. Cloudflare provides:

- DDoS protection (automatic, always-on)
- TLS 1.2/1.3 with HTTP/2
- Bot management (free tier)
- IP reputation filtering

The Cloudflare Tunnel (`cloudflared`) means the host has **no open inbound ports** — the attack surface is limited to what Cloudflare exposes.

## Security Headers

nginx adds the following headers on all responses:

- `X-Frame-Options: SAMEORIGIN`
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-Forwarded-Proto: https` (injected by nginx, not Cloudflare)

The real client IP is forwarded via the `CF-Connecting-IP` header from Cloudflare. nginx passes this through to the application containers.

## GHCR Authentication

The Spark must be authenticated to GHCR to push images. Log in once (or after token rotation):

```bash
echo $GHCR_TOKEN | docker login ghcr.io -u USERNAME --password-stdin
```

For local builds loaded directly into Docker, no registry authentication is required at runtime.

## Secrets

All secrets (`DATABASE_URL`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `TUNNEL_TOKEN`, `JWT_SECRET`) are stored in `/opt/deckvault/.env.prod`:

- Owned by `deckvault:deckvault`
- Permissions `600` — readable only by the `deckvault` user
- Never committed to the repository

```bash
# Verify permissions
ls -la /opt/deckvault/.env.prod
# Expect: -rw------- 1 deckvault deckvault
```
