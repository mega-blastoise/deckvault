# Infrastructure

## Host Machine

DeckVault runs self-hosted on an NVIDIA DGX Spark connected to a home network via ethernet.

| Property | Value |
|----------|-------|
| Machine | NVIDIA DGX Spark |
| LAN IP (ethernet) | `10.0.0.11` |
| Public IP | `75.71.208.15` (Comcast, no CGNAT) |
| OS user | `deckvault` (uid=996, shell `/sbin/nologin`, in `docker` group) |
| Prod directory | `/opt/deckvault/` |

No inbound ports are open on the host. All external traffic enters through the Cloudflare Tunnel.

## Cloudflare Tunnel

The `cloudflared` container runs a persistent outbound connection to Cloudflare's edge. Cloudflare receives HTTPS requests for `deckvault.gg`, terminates TLS at the edge, and pushes the request down the tunnel to nginx.

| Property | Value |
|----------|-------|
| Tunnel ID | `2c46aae5-b852-44de-848d-70270744a156` |
| DNS (authoritative) | `ben.ns.cloudflare.com`, `elisabeth.ns.cloudflare.com` |
| TLS | Terminates at Cloudflare edge — nginx is HTTP-only internally |
| Token | Stored in `/opt/deckvault/.env.prod` as `TUNNEL_TOKEN` |

## systemd Unit

The production stack is managed by a systemd unit at `/etc/systemd/system/deckvault.service`. It runs as the `deckvault` user and sets `COMPOSE_PROJECT_NAME=pokemon` to ensure the existing `pokemon_postgres_data` Docker volume is reused across restarts.

```bash
# Start / stop / restart
sudo systemctl start deckvault
sudo systemctl stop deckvault
sudo systemctl restart deckvault

# Enable auto-start on boot
sudo systemctl enable deckvault

# View logs
sudo journalctl -u deckvault -f
```

The unit type is `oneshot RemainAfterExit=yes` — `active (exited)` with `Result: success` is the expected status after a clean start.

## PostgreSQL

User data (accounts, decks, collections, meta reports) is stored in a containerized PostgreSQL 17 instance (`pokemon-postgres`) running inside the same compose stack. Data is persisted in the `pokemon_postgres_data` Docker named volume.

The `rest-api` container is the only service that connects to Postgres. It runs migrations automatically on startup.

## SQLite (Card Data)

Pokemon TCG card and set data lives in `pokemon-data.sqlite3.db`, mounted read-only into `rest-api` and `graphql-api` from `/opt/deckvault/database/`. This file is never written to at runtime.

<Callout variant="info">
  When a new TCG set is released, the SQLite file must be regenerated and redeployed. See [Rolling Out a New Card Set](/rollout-sqlite).
</Callout>

## GHCR

Application images are distributed via GHCR. The Spark pulls images locally during local builds — no push/pull to the registry is required for local deployments. The registry is used for CI builds and for sharing images.
