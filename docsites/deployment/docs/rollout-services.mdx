# Rolling Out a Service

Use this runbook when you need to deploy changes to one or more of the three application services: `web`, `rest-api`, or `graphql-api`. All commands run from the monorepo root on the Spark.

## Single Service Rollout

### web

```bash
# 1. Build the image
docker build -t ghcr.io/mega-blastoise/deckvault-web:latest \
  -f apps/web/Dockerfile .

# 2. Restart just the web container (no stack restart needed)
docker compose -f /opt/deckvault/docker-compose.prod.yml \
  up -d --no-deps web

# 3. Verify
docker ps --format "table {{.Names}}\t{{.Status}}" | grep pokemon-web
curl -sI https://deckvault.gg/ | head -3
```

`web` has no healthcheck — it starts immediately. `nginx` continues serving during the brief restart gap.

### rest-api

```bash
# 1. Build the image
docker build -t ghcr.io/mega-blastoise/deckvault-rest-api:latest \
  -f apps/rest-api/Dockerfile .

# 2. Restart rest-api (migrations run automatically on startup)
docker compose -f /opt/deckvault/docker-compose.prod.yml \
  up -d --no-deps rest-api

# 3. Wait for healthy, then verify
docker ps --format "table {{.Names}}\t{{.Status}}" | grep pokemon-rest-api
curl -s https://deckvault.gg/api/v1/health
```

<Callout variant="info">
  `rest-api` runs pending Postgres migrations on every startup. A rollout with new migrations will apply them automatically. Verify migration output in logs:
  ```bash
  docker logs pokemon-rest-api --tail=30
  ```
</Callout>

### graphql-api

```bash
# 1. Build the image
docker build -t ghcr.io/mega-blastoise/deckvault-graphql-api:latest \
  -f apps/graphql-api/Dockerfile .

# 2. Restart graphql-api
docker compose -f /opt/deckvault/docker-compose.prod.yml \
  up -d --no-deps graphql-api

# 3. Verify
docker ps --format "table {{.Names}}\t{{.Status}}" | grep pokemon-graphql-api
curl -s https://deckvault.gg/api/graphql -X POST \
  -H 'Content-Type: application/json' \
  -d '{"query":"{ __typename }"}'
```

## Rolling Out Multiple Services

Build all affected images first, then restart together:

```bash
# Build
docker build -t ghcr.io/mega-blastoise/deckvault-web:latest -f apps/web/Dockerfile .
docker build -t ghcr.io/mega-blastoise/deckvault-rest-api:latest -f apps/rest-api/Dockerfile .

# Restart both (--no-deps prevents cascading restarts of unaffected containers)
docker compose -f /opt/deckvault/docker-compose.prod.yml \
  up -d --no-deps web rest-api
```

## Logs

```bash
# Tail logs for any service
docker logs pokemon-web -f --tail=50
docker logs pokemon-rest-api -f --tail=50
docker logs pokemon-graphql-api -f --tail=50

# systemd unit logs (covers startup and compose output)
sudo journalctl -u deckvault -f
```

## Rollback

Docker doesn't retain previous `:latest` images once overwritten. Tag before deploying if you need a rollback target:

```bash
# Before building new image:
docker tag ghcr.io/mega-blastoise/deckvault-web:latest \
  ghcr.io/mega-blastoise/deckvault-web:rollback

# To roll back:
docker tag ghcr.io/mega-blastoise/deckvault-web:rollback \
  ghcr.io/mega-blastoise/deckvault-web:latest
docker compose -f /opt/deckvault/docker-compose.prod.yml \
  up -d --no-deps web
```
