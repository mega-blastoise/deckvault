# Containers & Services

Six containers are defined in `docker-compose.prod.yml`, all on the `pika` Docker network.

## pokemon-cloudflared

| Property | Value |
|----------|-------|
| Image | `cloudflare/cloudflared:latest` |
| Ports | None (outbound tunnel only) |
| Config | `TUNNEL_TOKEN` from `.env.prod` via `env_file` |

Initiates and holds an outbound tunnel to Cloudflare's edge. All inbound traffic for `deckvault.gg` arrives through this tunnel — no open inbound ports on the host are required. Depends on `nginx`.

## pokemon-nginx

| Property | Value |
|----------|-------|
| Image | `nginx:1.27-alpine` |
| Ports | `80:80` (host → container) |
| Config | `./docker/nginx/nginx.conf`, `./docker/nginx/conf.d/default.conf` |

The only container with an externally exposed port. Routes requests from the Cloudflare Tunnel to the three application containers. TLS terminates at Cloudflare — nginx is HTTP-only. Depends on `web` (started), `rest-api` (healthy), and `graphql-api` (healthy).

## pokemon-web

| Property | Value |
|----------|-------|
| Image | `ghcr.io/mega-blastoise/deckvault-web:latest` |
| Exposed port | `3000` (internal only) |
| Runtime | Bun SSR |

Environment variables injected at runtime:

```
PORT=3000
API_URL=http://rest-api:3001/api/v1
REST_API_URL=http://rest-api:3001
GRAPHQL_API_URL=http://graphql-api:3002
```

Communicates with both APIs over the `pika` network using Docker service names as hostnames. Never routes through nginx internally.

## pokemon-rest-api

| Property | Value |
|----------|-------|
| Image | `ghcr.io/mega-blastoise/deckvault-rest-api:latest` |
| Exposed port | `3001` (internal only) |
| Healthcheck | `wget -qO- http://localhost:3001/health` |
| Volume | `./database:/data` (read-only SQLite) |

Key environment variables:

```
REST_API_PORT=3001
REST_API_HOST=0.0.0.0
DATABASE_PATH=/data/pokemon-data.sqlite3.db
DATABASE_READONLY=true
DECK_DATABASE_PATH=/tmp/decks.sqlite3.db
```

Additional secrets (`DATABASE_URL`, `GOOGLE_CLIENT_ID`, etc.) come from `.env.prod` via `env_file`. Runs Postgres migrations automatically on startup. Depends on `postgres` (healthy).

## pokemon-graphql-api

| Property | Value |
|----------|-------|
| Image | `ghcr.io/mega-blastoise/deckvault-graphql-api:latest` |
| Exposed port | `3002` (internal only) |
| Healthcheck | `wget -qO- http://localhost:3002/health` |
| Volume | `./database:/data` (read-only SQLite) |

Key environment variables:

```
GRAPHQL_API_PORT=3002
GRAPHQL_API_HOST=0.0.0.0
DATABASE_PATH=/data/pokemon-data.sqlite3.db
DATABASE_READONLY=true
```

Reads only from SQLite — has no connection to Postgres.

## pokemon-postgres

| Property | Value |
|----------|-------|
| Image | `postgres:17-alpine` |
| Exposed port | `5432` (internal only) |
| Healthcheck | `pg_isready -U postgres -d pokemon` |
| Volume | `pokemon_postgres_data` (Docker named volume) |

Stores all user data (accounts, decks, collections, meta reports). The named volume `pokemon_postgres_data` persists across stack restarts and systemd unit restarts.

<Callout variant="warning">
  `COMPOSE_PROJECT_NAME=pokemon` is set in the systemd unit. This ensures Docker resolves the volume as `pokemon_postgres_data` regardless of which directory the compose command runs from. Do not change this.
</Callout>

## Startup Order

```
postgres (healthy)
    └── rest-api (healthy)
            └── web (started)
            └── nginx (started, after web + rest-api + graphql-api)
                    └── cloudflared (started, after nginx)
graphql-api (healthy, parallel with rest-api)
```

Healthchecks poll every 30 seconds with a 10 second timeout and 3 retries. `start_period` gives Bun services 15 seconds to boot before the first check.
