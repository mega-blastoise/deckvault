# Building

## All Workspaces

```bash
bun run build
```

Runs `turbo build` across the dependency graph. Packages build before the apps that depend on them.

## Individual Services

```bash
# Web
cd apps/web && bun run build

# REST API
cd apps/rest-api && bun run build

# GraphQL API
cd apps/graphql-api && bun run build
```

## Docker Images

Images are built from the monorepo root so Dockerfiles can `COPY` from workspace packages:

```bash
docker build -t ghcr.io/mega-blastoise/deckvault-web:latest \
  -f apps/web/Dockerfile .

docker build -t ghcr.io/mega-blastoise/deckvault-rest-api:latest \
  -f apps/rest-api/Dockerfile .

docker build -t ghcr.io/mega-blastoise/deckvault-graphql-api:latest \
  -f apps/graphql-api/Dockerfile .
```

Each Dockerfile is multi-stage: a build stage runs `bun install` and `bun run build`, then a lean final stage copies the output.

## Documentation Sites

```bash
# Build all docsites
bun run docs:build

# Build a single docsite
cd docsites/deployment && bun run docs:build
```

Output lands in `docsites/{name}/site/`.

## Type Checking

```bash
bun run check-types
```

Runs `tsc --noEmit` across all TypeScript workspaces. Run this before opening a PR.
